    개인정보처리방침 — Ploidy MCP 서버¶

    상태: 법률 자문 전 초안. GDPR(제13조), CCPA/CPRA, 개인정보보호법(PIPA), LGPD, 그리고 Claude.ai Connectors Directory 제출 요구사항에 구조적으로 정렬되어 있습니다. 구체적 문구·관할별 기준치·집행 가능성은 공개 전 자격 있는 법률 자문을 거쳐야 합니다.

    시행일: 2026-04-23 · 버전: 1.0-draft

    영문 원본: /docs/privacy-policy.md (해석 상충 시 영문 원본이 우선합니다.)

    0. 적용 범위¶

    본 방침은 다음에 적용됩니다.

    • 오픈소스 Ploidy MCP 서버 소프트웨어 자체.
    • 향후 공개될 유지관리자 직접 운영 배포본 (directory-submission.md의 URL).

    동일 소프트웨어를 사용하는 제3자 배포는 해당 운영자의 별도 방침이 적용되며, 본 문서는 소프트웨어 동작을 기술합니다.

    1. 개인정보처리자 / 개인정보보호책임자¶

    • 처리자: pyproject.toml에 표기된 유지관리자(heznpc).
    • 연락처: 보안 사안은 SECURITY.md. 일반 개인정보 문의는 레포지토리 GitHub Issues의 privacy 라벨 또는 Zenodo 레코드의 이메일.
    • 개인정보보호책임자(DPO) 지정: PIPA 제31조 의무지정 기준(공공기관·대규모 처리사업자) 미만으로 판단하여 별도 지정하지 않음. 규모가 기준을 넘으면 재검토.

    2. 수집·처리하는 개인정보 (PIPA 제15조·제17조)¶

    로컬 SQLite DB (PLOIDY_DB_PATH, 기본값 ~/.ploidy/ploidy.db, 0o700 소유자 전용 권한)에 저장:

    항목 필드 수집 출처 처리 목적 법적 근거
    디베이트 내용 debates.prompt, sessions.context_documents, sessions.compressed_summary (Semi-Fresh 요약), messages.content, convergence.synthesis, convergence.points_json, convergence.meta_analysis 이용자 제출 디베이트 실행 및 이력 제공 계약 이행 (GDPR §6(1)(b) / PIPA §15(1)2)
    디베이트 메타데이터 debates.id/status/config_json/created_at/updated_at, 세션 설정 자동 생성 라우팅·진단·재현성 정당한 이익 (GDPR §6(1)(f) / PIPA §15(1)6)
    테넌트 식별자 debates.owner_id — OAuth client_id 또는 bearer 토큰 레이블로 해석됨 OAuth/bearer 본인 디베이트에 대한 접근 인가·다중테넌트 격리 계약 이행
    OAuth 상태 (PLOIDY_AUTH_MODE=oauth 시) oauth_clients, oauth_codes(5분 단일 사용), oauth_tokens(32바이트 불투명 무작위 문자열, 액세스 1시간 / 리프레시 회전) OAuth 등록 RFC 6749 구현 계약 이행
    집계 지표 Prometheus 카운터 파생 운영·용량 계획 정당한 이익

    SQLite DB 외부로, ASGI 기본 로깅을 통해 서버 프로세스의 표준 출력(stdout)에 요청 경로·상태 코드·클라이언트 IP 주소가 기록될 수 있습니다. Ploidy는 이를 DB에 저장하지 않으며, 로그 샤핑을 구성한 운영자는 해당 로그에 대해 별도 개인정보처리자가 됩니다.

    3. 수집하지 않는 개인정보¶

    • 성명·이메일·전화번호·주소·결제정보.
    • 인구통계·생체정보·정밀 위치·민감정보 / 건강정보.
    • 만 14세 미만 아동(PIPA 제22조의2) / 만 13세 미만(COPPA) / GDPR 기본 16세 미만의 개인정보 — 본 서비스는 아동을 대상으로 하지 않음.
    • 모델 제공자 자격증명. PLOIDY_API_KEY / ANTHROPIC_API_KEY는 프로세스 환경변수에만 존재, DB에 기록되지 않음.
    • 행동 광고 신호. 분석 도구·광고 네트워크·쿠키·핑거프린팅· 크로스 사이트 트래커 일체 미사용.

    4. 제3자 데이터 흐름 (PIPA 제17조·제28조의8)¶

    Ploidy는 이용자가 제출한 텍스트를 운영자가 PLOIDY_API_BASE_URL로 지정한 하나의 LLM 제공자에게 전송합니다. 기본값은 Anthropic의 OpenAI 호환 엔드포인트: https://api.anthropic.com/v1/openai.

    제공자에게 전송되는 데이터: 디베이트 프롬프트, 컨텍스트 문서, 응답 도출에 필요한 시스템 프롬프트 메타데이터. Ploidy를 떠난 이후의 처리는 제공자의 방침이 적용됩니다(Anthropic의 상업적 이용 약관 및 개인정보 처리방침).

    Ploidy는 다른 제3자에게 개인정보를 판매·임대·공유하지 않으며, 광고 파트너·데이터 브로커·분석 처리자가 없습니다.

    5. 국외 이전 (PIPA 제28조의8, GDPR §44-50)¶

    기본 제공자 엔드포인트는 미국에 호스팅되어 있습니다. EEA·영국 또는 국외 이전 규제가 있는 관할의 이용자가 제출한 데이터는 미국으로 이전됩니다. Anthropic은 EU 고객 대상 Standard Contractual Clauses를 제공하므로 사용 전 적격성 확인이 권장됩니다. 지역 엔드포인트가 제공되면 운영자가 PLOIDY_API_BASE_URL을 변경할 수 있습니다.

    한국 이용자 대상: 제출된 프롬프트 및 컨텍스트는 상기 모델 제공자에게 이전됩니다. 기본 구성으로 본 서비스를 이용하는 것은 PIPA 제28조의8 상 해당 이전에 대한 동의로 간주됩니다(디베이트 실행 목적). 동의하지 않는 경우 한국 체류 엔드포인트를 구성한 운영자의 배포본을 이용해야 합니다.

    6. 보유·파기 기간¶

    데이터 기본 보유 기간
    완료/취소된 디베이트 이용자 또는 운영자의 삭제 시까지, 또는 PLOIDY_RETENTION_DAYS(기본값 0 = 비활성) 자동 파기
    활성/일시중단 디베이트 완료/취소 전에는 파기 대상 아님
    OAuth 인가 코드 5분 또는 1회 사용 중 먼저 도달 시
    OAuth 액세스 토큰 1시간
    OAuth 리프레시 토큰 사용 시마다 회전; /revoke 명시 호출 시 즉시 폐기
    Prometheus 카운터 메모리 내; 서버 재시작 시 초기화

    파기 방법:

    • ploidy-history delete <id> CLI (예정) / debate_delete MCP 툴 — 개별 디베이트 단위.
    • PLOIDY_DB_PATH 파일 삭제 — 전체 데이터.
    • OAuth /revoke 엔드포인트 — 특정 토큰.

    7. 정보주체 권리 (PIPA 제35~37조, 제39조)¶

    EEA(GDPR §15-22), 영국(UK GDPR), 캘리포니아(CCPA/CPRA), 한국(PIPA §35-37, §39), 브라질(LGPD §18) 및 동등 권리를 부여하는 관할 모두에서 아래 권리가 적용됩니다.

    권리 행사 방법
    열람·처리 확인 자체 호스팅 시 ploidy-history show <id>; 호스팅 배포 이용 시 운영자 문의(1항)
    정정 디베이트 출력은 모델 산출물의 기록이므로 "정정" 대상이 아님. 삭제 후 재실행. OAuth client_name·redirect URI는 재등록
    삭제·파기 개별 디베이트는 debate_delete; 전체는 DB 파일 삭제
    처리 제한 OAuth 토큰 /revoke로 폐기, 신규 디베이트 제출 중단
    이동성(portability) SQLite DB 파일은 표준 SQLite 클라이언트로 읽기 가능
    반대(GDPR §21) 처리자에게 이메일; 정당한 이익 기반 처리를 요청 시 중단
    프로파일링·자동화된 결정(GDPR §22) Ploidy는 인간 판단 지원 도구이며, 법적·유사 효과를 갖는 자동화된 결정을 수행하지 않음
    CCPA "판매·공유 금지" 해당 없음 — Ploidy는 교차 맥락 행동 광고 목적으로 개인정보를 판매·공유하지 않음
    신고 권리 EEA: 관할 감독기관. 영국: ICO. 한국: 개인정보보호위원회. 캘리포니아: 주 법무장관

    호스팅 배포 운영자는 이러한 권리 행사 채널을 제공해야 합니다. 유지관리자 직접 운영 배포본은 가능한 범위에서 30일 이내 응답합니다(GDPR §12(3)).

    8. 보안¶

    • 기본 bearer 토큰 인증(PLOIDY_TOKENS)은 상수 시간 비교 (hmac.compare_digest) 사용.
    • OAuth 2.0 모드(PLOIDY_AUTH_MODE=oauth)는 PKCE S256, 단일 사용 인가 코드, 리프레시 토큰 회전 강제.
    • DB 파일은 0o700 (소유자 전용) 권한으로 생성.
    • 테넌트별 속도 제한(PLOIDY_RATE_CAPACITY, PLOIDY_RATE_PER_SEC) 제공.
    • 액세스 토큰은 32바이트(256비트) 불투명 무작위 문자열로 DB에 저장 — 해시 아님, 서명 아님. 방어선은 폐기(revocation).

    완벽한 보안은 없습니다. 취약점은 SECURITY.md로 비공개 신고.

    9. 아동의 개인정보¶

    Ploidy는 만 14세 미만(PIPA 제22조의2) / 만 13세 미만(COPPA) / GDPR 기본 16세 미만 아동을 대상으로 하지 않으며, 해당 아동의 개인정보를 고의적으로 수집하지 않습니다. 운영자가 아동 개인정보 수집을 인지한 경우 즉시 삭제해야 합니다.

    10. 지역별 고지¶

    10.1 캘리포니아(CCPA/CPRA)¶

    • 수집 카테고리: Identifiers, Internet/electronic network activity, Inferences.
    • 출처: 이용자 직접 제출.
    • 업무·상업적 목적: 서비스 제공, 보안 유지, 디버깅.
    • 공유·판매: 해당 없음 — "Do Not Sell or Share My Personal Information" 링크는 활성화 조건이 충족되지 않아 요구되지 않음.
    • 정정권: 삭제 후 재실행 / OAuth 재등록으로 행사.
    • 민감 개인정보 제한권: Ploidy는 §1798.140(ae)에 정의된 민감 개인정보를 수집하지 않음.

    10.2 EEA / 영국(GDPR / UK GDPR)¶

    • 처리별 법적 근거는 §2에 명시.
    • 국외 이전은 §5 참조.
    • 자동화된 결정 미수행(§7).
    • 감독기관 신고 가능(영국: ICO). 정당한 이익 평가 (Legitimate Interests Assessment)는 요청 시 제공.

    10.3 한국(개인정보보호법)¶

    • 처리 목적·보유 기간·수집 항목: §2, §3, §6.
    • 국외 이전: §5. 기본은 Anthropic 미국 엔드포인트.
    • 정보주체 권리(제35~37조, 제39조): §7. 1항 채널로 행사.
    • DPO(개인정보보호책임자): 의무 지정 기준 미만으로 미지정, 유지관리자가 업무 대행. 규모 도달 시 지정.
    • 한국어 고지: 본 문서가 한국어 번역본. 영문 원본과 해석 상충 시 영문이 우선(§0).

    10.4 브라질(LGPD)¶

    포르투갈어 번역은 현재 유지되지 않습니다. LGPD 제18조 권리는 §7과 동일하며 §1 채널로 행사.

    10.5 기타 관할¶

    본 방침은 위 체제의 최대공약수로 작성되었습니다. 인도 DPDP Act, 중국 PIPL, 캐나다 PIPEDA 등이 추가 권리를 부여하는 경우 §7 채널을 통해 합리적 범위에서 존중합니다.

    11. 민감 범주 안전장치¶

    Ploidy는 행동 광고 목적 프로파일링, 생체 추론, 건강 진단, 법률·의료 자동 조언을 일절 수행하지 않습니다. 의료·법률· 금융 내용의 제출은 이용자 책임이며 서비스 의도 범위 밖입니다 (이용약관 §3 참조).

    12. 변경¶

    중대한 변경은 본 파일에 커밋되어 git 이력을 통해 감사 가능합니다. 상단의 시행일과 버전이 매 중대 변경 시 업데이트됩니다.

    13. 해석 관할¶

    본 방침은 유지관리자 직접 운영 배포본에 대해 대한민국법에 따라 해석됩니다. 지역 감독기관은 거주민에 대해 법정 권한을 유지합니다.

    이 문서는 법률 자문 전 초안입니다. 독립적 검증 없이 최종 법률 자문으로 신뢰하지 마십시오.
