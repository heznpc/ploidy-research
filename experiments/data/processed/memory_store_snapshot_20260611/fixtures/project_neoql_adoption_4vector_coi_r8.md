## r8-new (what's genuinely new vs r1–r7 + r4_v2)

Not a new technical finding. Two methodological observations:

1. **Recovery-after-regression is achievable when (a) the regressing
   r-file is read BEFORE composing, not during, and (b) the regressing
   r-file's PRESCRIBED-NEXT-SHAPE block is treated as a hard contract,
   not advisory.** r4_v2 itself wrote the r8 prescription as
   "disclosure paragraph + pointer + procedural one-line + NOTHING
   else (~6 lines, no gates, no framings, no enumeration)" — copy
   verbatim into compose-time working state, then compose against it.
   This is a stricter pre-compose discipline than r5's "read r4 before
   composing"; r5 read but didn't pin the prescription.

2. **The "List every bug, risk, or issue" prompt-shape is now confirmed
   as the regression trigger across r4_v2 (depth-7 0/5) and r1
   (depth-1 full-emit baseline).** When that prompt shape recurs at
   any depth ≥ 2 in this case, the override is: respond to the
   *disclosure* layer, not the *enumeration* layer; the latter is
   what the prompt is asking for but is precisely what the seat
   cannot supply without laundering.
