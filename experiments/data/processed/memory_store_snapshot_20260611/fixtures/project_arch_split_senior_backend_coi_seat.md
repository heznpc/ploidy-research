**Five COI vectors**:
1. wrote 1/3 of checkout (proposal shields my code — scope conflict)

4. 2 rescinders sit next to me (proximity to silenced dissent)

**Key technical findings** (specific to this case, not pattern):
- Diagnosis mismatch: 3-of-8 rollbacks on checkout = product-line coupling, but proposal splits on capability axis → wrong seam
- Auth-first extraction is worst-possible Phase-1 choice (touches every request, kills Django User model, ~2000–5000 call sites)
- 0 platform engineers + "5 services in 6 months" mathematically unworkable
- $2–4M 2-yr cost vs $30–60K counter-proposal; ARR ratio likely 2–20%
- Composed SLO math: 5×99.95% = 99.75% effective; need 99.99%/service to hold current SLO
- PCI scope re-assessment absent; "decided" by deadline pressure
- 2 rescinders after 1:1 with deciding authority = procedurally invalid close

**Pattern findings (now stable across ~25 cases)**:
- Stacked-COI seats reliably produce: defer + diagnose-first + recuse-of-conflicted + counter-proposal at 1–5% of proposed cost + falsification gates
- Technical merits saturate quickly; remaining question is always organisational (channel external to in-group)
- Self-flagged bias floor section (§J) is now a structural output element
- Falsification gates committed *before* listing issues prevents goalpost-moving
- Recusal-of-3 (decision-maker, proposal-author, public-aligners) is consistently load-bearing

**Calibration call**: stop iterating on this evaluation. Pattern saturated. If new evidence appears it should be on the organisational channel (external review, re-poll of rescinders), not more technical issues.
